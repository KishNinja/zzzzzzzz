Объектно-ориентированное программирование (ООП) предоставляет концепцию классов и объектов. Давайте рассмотрим основные понятия в рамках ООП:

Классы :

  • Определение : Класс - это шаблон или чертёж для создания объектов. Он определяет атрибуты (переменные) и методы (функции), которые будут присутствовать у созданных объектов.

    Пример :

    • В приведенном примере Car - это класс, который имеет атрибуты make, model, и year, а также метод start_engine.

Объекты :

  • Определение : Объект - это экземпляр класса. Это конкретное воплощение данных и функциональности, определенных в классе.

    Пример :

    • В этом примере my_car - это объект класса Car. Мы создаем экземпляр класса Car с конкретными значениями атрибутов.

  • Методы : Методы - это функции, связанные с объектами. Они предоставляют функциональность, связанную с объектом.

Инкапсуляция, наследование и полиморфизм :

  • Инкапсуляция : Скрытие деталей реализации объекта и предоставление интерфейса для взаимодействия с ним.

  • Наследование : Возможность создания нового класса на основе существующего, наследуя его свойства и методы.

  • Полиморфизм : Возможность использовать объекты схожих типов с единым интерфейсом.

    • ООП предоставляет удобный способ организации кода, повышает его читаемость и обеспечивает повторное использование кода.
